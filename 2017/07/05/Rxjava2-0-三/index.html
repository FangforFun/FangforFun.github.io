简单的时序demo(timer, interval, delay)

这是一个超级简单明了的例子展示了如何使用RxJava的timer，interval和delay运营商来处理一堆案件要在特定时间间隔运行任务。对Android基本上用TimerTask说不。

  1. 在延迟2秒后运行一个任务，然后完成
  2. 每1秒不间断运行任务（在第一个任务启动之前有1秒的延迟）
  3. 每1秒不间断运行任务（与上述相同但在第一个任务启动之前没有任何延迟）
  4. 每3秒运行一次任务，但运行5次后，自动终止
  5. 运行任务A，暂停一段时间，然后执行任务B，然后终止

指数回退(retry, delay)

指数退避是一种策略，其基于来自某个输出的反馈，我们改变进程的速率（通常减少重试次数或在重试或重新执行某个进程之前增加等待时间）。

这个概念比较有道理。RxJava使得（相对）简单地实现这样的策略。

  1. 出错重试retry

说你有一个网络故障。一个明智的策略是不要每隔1秒重试一遍你的网络呼叫。这将是聪明的（不，优雅！）以越来越多的延迟重试。所以你尝试在第二次执行网络通话，没有骰子？尝试10秒…否定？20秒后尝试，没有cookie？1分钟后尝试。如果这件事仍然失败，你必须放弃网络哟！

我们使用RxJava与retryWhen运算符模拟此行为。

  1. delay变体

执行给定次数的操作,每次操作的延迟间隔递增;执行完给定的四次操作停止执行;通过delay运算符;
